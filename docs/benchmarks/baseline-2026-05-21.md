# Safe Zone benchmark baseline

Date: 2026-05-21

Host:
- OS: Windows
- CPU: 11th Gen Intel(R) Core(TM) i7-1185G7 @ 3.00GHz
- Go target: `windows/amd64`

Command:

```sh
go test ./internal/analysis ./internal/risk -run "^$" -bench . -benchmem
```

Results:

```text
BenchmarkAnalyzeSafeDomain-8          75064 ns/op    6368 B/op   134 allocs/op
BenchmarkAnalyzeSuspiciousDomain-8   291997 ns/op   16504 B/op   140 allocs/op
BenchmarkAnalyzeNoRedis-8            245966 ns/op   16882 B/op   149 allocs/op
BenchmarkWhitelistLookup-8              201.1 ns/op    64 B/op     2 allocs/op
```

Interpretation:
- Lexical-only analysis is comfortably below 1 ms per domain on this development machine.
- The no-Redis service path is still below 1 ms for the benchmarked suspicious domain because network enrichment and AI are not enabled in this benchmark.
- This does not prove the SRS target of 500 qps cache-hit / 50 qps full-miss on Oracle ARM. That needs a VPS benchmark with Redis, DoH, TLS/WHOIS, and optional Ollama enabled.
